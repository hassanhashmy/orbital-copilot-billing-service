# Orbital Copilot Billing Service

A Python API that calculates credit usage for each message in the current Orbital Copilot billing period. It fetches message and report data from upstream services, applies text-based pricing rules when a report is unavailable, and returns a unified usage summary.

## Running the Solution

### Prerequisites

- Python 3.11+

### Setup

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

An `.env.example` file is provided. Copy it to `.env` if you want to override the default upstream API base URL:

```bash
cp .env.example .env
```

### Start the Server

```bash
uvicorn app.main:app --reload
```

The API will be available at `http://127.0.0.1:8000`.

- **GET /usage** — returns credit usage for the current billing period
- **GET /docs** — interactive Swagger UI documentation (auto-generated by FastAPI)

### Run Tests

```bash
python -m pytest tests/ -v
```

All 46 tests should pass (40 unit tests for credit calculation, 6 integration tests for the endpoint).

## Design Decisions

**FastAPI** was chosen because the endpoint needs to call up to 8 report URLs per request. FastAPI's native `async/await` support lets me fire those calls concurrently with `asyncio.gather`, and its built-in Pydantic integration provides type validation at the data boundary with no extra wiring. The auto-generated Swagger UI at `/docs` is a bonus for reviewers.

**Layered folder structure.** Code is organised by responsibility — `clients/` for upstream HTTP communication, `services/` for business logic, `routes/` for request handling, `models/` for schemas, and `core/` for cross-cutting concerns like config and dependency injection. The folder tells you the layer, the filename tells you the domain.

**Pure credit calculator module.** The seven pricing rules are implemented as pure functions with no HTTP or framework dependencies. This made it possible to write 40 unit tests for the calculation logic without needing any mocks, and to develop the pricing rules independently of the API layer.

**OrbitalCopilotClient class.** All upstream API communication is encapsulated in a single class with a generic `_get()` method. The base URL is loaded from environment configuration, and domain-specific methods (`fetch_current_period_messages`, `fetch_report_by_id`) handle parsing and 404 fallback. Adding a new upstream endpoint would be one method.

**Dependency injection for the HTTP client.** The `OrbitalCopilotClient` is provided via FastAPI's `Depends()`, which manages the `httpx.AsyncClient` lifecycle per request and makes it trivial to substitute a mock in tests using `dependency_overrides`.

**Concurrent report fetching with de-duplication.** Before fetching reports, unique report IDs are collected into a set. `asyncio.gather` fetches them all in parallel, and results are stored in a dictionary. This avoids redundant HTTP calls (the same report ID can appear on many messages) and minimises latency.

**`response_model_exclude_none=True`.** When a message has no associated report, the `report_name` field is omitted from the JSON output entirely rather than appearing as `null`. This matches the spec: "if there is one, otherwise this field should be omitted."

**Environment configuration via python-decouple.** The upstream API base URL is read from a `.env` file (with a sensible default), keeping environment-specific values out of the codebase.

## Assumptions & Edge Cases

- **Word definition**: a word is a maximal contiguous sequence from the character set `[a-zA-Z'-]`. Contractions like `don't` and hyphenated terms like `well-known` are treated as single words.
- **Unique word bonus**: requires at least one word; an empty message does not receive the bonus.
- **Palindrome check**: an empty string is not treated as a palindrome.
- **Rule ordering**: the minimum cost floor of 1 credit is applied after the unique word bonus but before the palindrome doubling. A palindrome with unique words can therefore have a minimum cost of 2 credits.
- **404 report fallback**: when a message has a `report_id` but the reports API returns 404, credits are calculated from the message text and `report_name` is omitted.
- **Floating point**: all credit components are multiples of 0.05, so the final result is rounded to 2 decimal places to eliminate floating-point artefacts.

## Error Handling
If the upstream messages or reports APIs are unreachable or return non-404 errors, the endpoint returns HTTP 502 with a descriptive error message. The 404 case for reports is handled gracefully as a fallback to text-based credit calculation, not as an error.
